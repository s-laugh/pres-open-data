Opening Our Data Up

What is the simplest technical way to open our data to the public?

Post our DB passwords.

Okay... we can make the account read only.

Challenges

Interoperability

If the data is open but no one can understand it, it's useless

  • Data Categorization
  • Data Flows - Ownership

Availability

Who can access what data, how do we know and manage it

  • Security / Authentication
  • Legacy Systems Integration

Common methods

  • Data Dumps
  • APIs

Options

Data Dumps

Build up a collection of data, and dump it somewhere on the internet (ex. reports).

Benefits

  • Vetting the data
  • Single point for access

Risks

  • One time deal (maintainability)
  • Unknown Consumers
  • Ambiguity of presented data (big brother?)

Open APIs

Have the groups that own an operate data, expose it though APIs

Benefits

  • Data ownership and knowledge stays localized
  • Integration points can be customized
  • Scaling is simplified

Risks

  • Varying Interoperability
  • Onboarding
My Recommendation

Using Open APIs

Why?

  • Sharing & Collaborating
  • Security & Monitoring Controls
  • Scaling & Resources Management

Starting a Pilot

Ensure it addresses all the concerns.

Start small.

Interoperability

We can't force a common standard of the data, but we can require the API is interpretable by:

  • Require data be categorized
  • Require API identify the data formatting

We then put this information into a data-catalog that is searchable.

Availability

This is essentially solved by the API. APIs provide:

  • Secure Access control, to the API and the data sent based on user (client concent)
  • Integration is customizable for each API with any system.

Getting Started

  1. Start with 1-2 willing groups.
    • public data
    • concent based data
  2. Identify the data we are looking to open up
  3. Find 3rd parties interested in consuming the data - work with them as clients
  4. Start exposing small portions of data over time
  5. Expand
